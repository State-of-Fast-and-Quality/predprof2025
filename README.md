# predprof2025

Решение команды **Царицынская Братва (State of Fast and Quality)** для **Московской предпрофессиональной олимпиады школьников 2024/2025**.

Проект посвящён задаче анализа поверхности Марса, сборки карты по тайлам, поиску подходящих точек размещения базовых станций и визуализации результатов в интерактивном веб-интерфейсе.

## О проекте

В основе решения лежит связка из нескольких компонентов:

- получение данных с внешнего API;
- сборка карты Марса из отдельных тайлов;
- сохранение данных в PostgreSQL;
- анализ рельефа и поиск ключевых точек;
- визуализация карты, модулей связи, станций и зон покрытия;
- интерактивный интерфейс на **Gradio**.

Репозиторий показывает не только алгоритмическую часть решения, но и полноценную инженерную упаковку: от получения данных до интерактивной демонстрации результата.

## Постановка задачи

1. Из API поступают фрагменты карты Марса.
2. Эти фрагменты собираются в единую карту поверхности.
3. Отдельно запрашиваются координаты двух модулей:
   - отправителя,
   - получателя.
4. Анализируется рельеф поверхности.
5. По найденным высотным точкам выбираются позиции для базовых станций.
6. Рассчитываются зоны покрытия и итоговая стоимость сети.

Таким образом, задача сочетает в себе:

- работу с внешним API,
- обработку матриц и карт высот,
- элементы геоаналитики,
- хранение промежуточных данных в БД,
- построение визуального интерфейса.

## Сильные стороны решения

Проект выделяется тем, что это не просто исследовательский ноутбук, а **законченное прикладное решение**:

- есть backend-логика для получения и обработки данных;
- используется база данных для хранения результатов;
- реализован пользовательский интерфейс;
- визуализация делает решение наглядным;
- архитектура хорошо подходит для демонстрации на олимпиаде или защите проекта.

Это даёт проекту сильную инженерную составляющую и показывает умение работать не только с алгоритмом, но и с целостным продуктом.

## Архитектура решения

### 1. Получение данных с API
Из внешнего API загружаются:

- тайлы карты поверхности;
- координаты модулей связи;
- стоимость станций разных типов.

### 2. Сборка карты
Карта формируется из тайлов размером `64×64`, которые объединяются в итоговую матрицу `256×256`.

### 3. Хранение данных
Для хранения используются таблицы PostgreSQL:

- карта поверхности;
- данные о модулях;
- данные о станциях.

Это позволяет сохранять результаты обработки и упрощает дальнейший анализ.

### 4. Анализ рельефа
На карте ищутся локальные пики высоты. Эти точки используются как кандидаты для размещения станций.

### 5. Построение сети
Для найденных точек задаются станции двух типов:

- **Cuper**
- **Engel**

Для них учитываются:

- координаты,
- радиус покрытия,
- стоимость размещения.

### 6. Визуализация
В интерфейсе доступны:

- 3D-визуализация карты;
- отображение модулей;
- отображение станций;
- отображение зон покрытия;
- расчёт количества станций и общей стоимости.
